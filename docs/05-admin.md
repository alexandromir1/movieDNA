# 05. Admin Panel

---

# Purpose

Административная панель предназначена для создания, тестирования, публикации и анализа ежедневных Challenge.

MovieDNA является контентно-ориентированным продуктом. Качество пользовательского опыта напрямую зависит от качества создаваемых Challenge.

Главная задача административной панели — максимально сократить время создания нового Challenge при сохранении высокого качества.

### Основные принципы

- Простота важнее количества функций.
- Каждое действие должно сокращать время подготовки нового Challenge.
- Один полностью готовый Challenge должен создаваться не более чем за 5 минут (при наличии готового изображения).
- Админка помогает создавать качественные загадки, а не просто хранить данные.
- Все проверки качества должны выполняться до публикации.

# Challenge Lifecycle

Каждый Challenge проходит ручной жизненный цикл подготовки.

```text
draft
↓

ready
↓

scheduled
```

Ручные статусы:

| Status | Значение |
|--------|----------|
| draft | Черновик, в работе |
| ready | Полностью готов, ещё не в расписании |
| scheduled | Запланирован; дата публикации задана |

Поле `date` = publishAt (UTC YYYY-MM-DD).

Вычисляемые состояния (не хранятся в JSON):

| Bucket | Правило |
|--------|---------|
| Upcoming | scheduled + date > today |
| Today's Challenge | scheduled + date == today |
| Archive | scheduled + date < today |

Дата публикации — единственный источник правды для показа Daily Challenge и архива.

Dashboard является главным экраном административной панели.

Он показывает текущее состояние проекта.

Отображается:

- следующий Challenge;

- количество запланированных публикаций;

Главная цель Dashboard — за несколько секунд показать текущее состояние проекта.

# Challenge Editor

Challenge Editor является основным рабочим инструментом.

Экран разделен на две части.

## Левая часть

Большое рабочее изображение.

Именно здесь происходит создание Reveal Regions.

## Правая часть

Панель настроек Challenge.

Содержит:

### Основная информация

- Название (RU)
- Название (EN)
- Год выхода
- Жанры
- Страны
- Продолжительность
- Теги

### Настройки

- дата публикации;
- статус;
- доступность в архиве.

# Reveal Region Editor

Reveal Region Editor является частью Challenge Editor.

Отдельного экрана не существует.

Редактор позволяет:

- создавать Reveal Region;
- изменять размер;
- перемещать;
- изменять форму;
- удалять;
- изменять порядок открытия.

Reveal Regions создаются непосредственно поверх изображения.

Поддерживаются инструменты:

- Rectangle
- Ellipse
- Polygon

Количество Reveal Regions в MVP всегда равно пяти.

Изменение количества невозможно.

---

# Reveal Region Rules

Reveal Regions должны соответствовать следующим правилам.

- Reveal Region 1 — самая слабая подсказка.
- Каждая следующая Reveal Region должна увеличивать вероятность правильного ответа.
- Reveal Region 5 должна практически гарантировать узнаваемость фильма.
- Reveal Regions не должны перекрываться.
- Reveal Regions не должны выходить за пределы изображения.

Порядок Reveal Regions определяет стоимость открытия.

Редактор не может изменять стоимость отдельных Reveal Regions.

# Preview Mode

Каждый Challenge обязательно проходит тестирование перед публикацией.

Preview полностью повторяет пользовательский режим.

В режиме Preview доступны:

- поиск фильма;
- Reveal Regions;
- скрытый таймер;
- подсчет Movie Score;
- итоговый экран.

Preview используется исключительно для проверки качества Challenge.

---

# Publish Checklist

Перед публикацией система автоматически выполняет проверку.

Challenge может быть опубликован только при успешном прохождении всех пунктов.

Проверяется:

- заполнено название RU;
- заполнено название EN;
- загружено изображение;
- создано пять Reveal Regions;
- отсутствуют пересечения Reveal Regions;
- дата публикации свободна;
- Preview успешно завершен.

При наличии ошибок публикация невозможна.

---

# Publication

Challenge может быть:

- опубликован немедленно;
- запланирован;
- снят с публикации.

После публикации Challenge автоматически становится ежедневной игрой.

---

# Archive

После окончания дня Challenge автоматически переносится в архив.

Архивные Challenge сохраняют:

- изображение;
- Reveal Regions;
- статистику;
- дату публикации.

Архив остается доступным игрокам.

---

# Validation

Перед публикацией редактор должен подтвердить соответствие Challenge требованиям проекта.

Проверяется:

- изображение соответствует художественному стилю проекта;
- отсутствуют кадры фильма;
- отсутствуют лица актеров;
- отсутствуют элементы, нарушающие авторские права;
- изображение передает атмосферу фильма;
- Reveal Regions расположены от слабой подсказки к сильной;
- Challenge соответствует требованиям документа 04b-level-design.md.

- количество игроков;

# Design Philosophy

Административная панель должна помогать создавать качественные ежедневные Challenge.

Она не должна перегружать редактора настройками.

Если качество Challenge можно улучшить за счет дизайна загадки, это предпочтительнее, чем добавление новых параметров в систему.

Простота рабочего процесса важнее количества функций.

Все сложные инструменты добавляются только после появления статистики и подтвержденной необходимости.
