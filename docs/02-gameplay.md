Purpose

Describe how the player interacts with MovieDNA.
1. Цель документа

Этот документ описывает все игровые механики MovieDNA, пользовательские сценарии и состояния игры.

Gameplay является основой продукта. Любые изменения игрового процесса должны сначала вноситься в этот документ.

2. Core Gameplay Loop

Главный цикл игры.

Открыть сайт
        │
        ▼
Today's Challenge
        │
        ▼
Reveal Region (текущая)
        │
        ▼
Ввести ответ (Guess)
        │
 ┌──────┴────────┐
 │               │
 ▼               ▼
Верно          Неверно
 │               │
 ▼               ▼
Победа     Автоматически открывается
 │         следующая Reveal Region
 │         (штраф Movie Score)
 │               │
 └──────┬────────┘
        ▼
Следующая попытка

Дополнительно игрок может вручную нажать кнопку открытия следующей Reveal Region без ввода ответа.

Если все Reveal Regions уже открыты, неверный ответ не открывает ничего нового и не начисляет дополнительный штраф. Игрок может продолжать вводить ответы.

3. Архитектура контента

В проекте разделены три сущности.

Movie
Информация о фильме: название, оригинальное название, год, aliases.

Level
Игровой контент: изображение, Reveal Regions, accepted answers, сложность.

Challenge
Публикация конкретного Level на конкретную UTC-дату.

Один Movie может использоваться в нескольких Level.
Один Level публикуется как Challenge на дату.

4. Состояния игры (Game States)

Это один из важнейших разделов.

NOT_STARTED

↓

IN_PROGRESS

↓

COMPLETED

↓

ARCHIVED
NOT_STARTED

Игрок еще не начал сегодняшнюю игру.

Доступно:

посмотреть изображение;
открыть первую подсказку;
ввести ответ.
IN_PROGRESS

Игра уже началась.

Хранятся:

открытые Reveal Regions;
использованные подсказки;
время;
количество попыток.
COMPLETED

Фильм угадан.

Показываются:

очки;
статистика;
кнопка Share;
переход в архив.
ARCHIVED

Игра находится в архиве.

Повторное прохождение не влияет на основной рейтинг.

5. Структура Level

Каждый Level состоит из:

id
movieId
difficulty
image
width
height

acceptedAnswers[]
metadata

Каждая Reveal Region:

id
name
polygon
displayOrder (1–5)
scorePenalty

Во всех Challenge используется ровно 5 Reveal Regions.

Экран состоит из:

Количество очков
Индикатор открытых областей
7. Ввод ответа

Игрок вводит название фильма.

Поддерживаются:

русский язык;
английский язык.

Например:

Terminator Two

↓

Терминатор 2

↓

Принимается

Примечание: Для каждого фильма хранится официальный список допустимых ответов на русском и английском языках. Допустимые варианты определяются администратором при создании игры.

8. Проверка ответа

Ответ считается правильным если:

совпадает полностью;
совпадает с alias;
проходит проверку на небольшие опечатки (например, расстояние Левенштейна);
игнорируется регистр.

Не учитываются:

лишние пробелы;
дефисы;
разные варианты написания.
9. Reveal System

Основная механика игры.

Изображение изначально скрыто.

Игрок открывает Reveal Regions по очереди.

Каждая Reveal Regions имеет:

id
polygon
displayOrder
scorePenalty
animation

10. Image Viewer

Статус: Version 1.1

В MVP изображение отображается без масштабирования и перемещения.

Планируется в Version 1.1:


Игрок может свободно исследовать изображение во время прохождения.

Desktop
увеличение колесиком мыши;
перетаскивание изображения;
двойной клик для быстрого увеличения;
кнопка «Сбросить масштаб».
Mobile
pinch-to-zoom;
перетаскивание изображения;
двойной тап для быстрого увеличения;
инерционная прокрутка.
Ограничения
Масштаб не влияет на игровой процесс.
Все открытые Reveal Regions сохраняют корректное положение независимо от масштаба изображения.
После обновления страницы масштаб сбрасывается, а прогресс открытия областей сохраняется.

11. Reveal Order

Reveal Regions открываются строго в заданном порядке.

Пользователь не может выбирать следующую Reveal Region самостоятельно.

(Мы можем обсудить позже, не станет ли режим со свободным выбором интересным дополнительным режимом.)

12. Подсказки

Подсказкой считается открытие следующей Reveal Region.

Текстовые подсказки отсутствуют.

Во всех Challenge ровно 5 Reveal Regions:

Reveal Regions 1–4 — отдельные области изображения;
Reveal Region 5 — полное изображение.

13. Попытки

Неверный ответ автоматически открывает следующую Reveal Region (если она ещё не открыта) и уменьшает потенциальный Movie Score.

Штраф начисляется только за открытие новых Reveal Regions, не за сам факт ошибки.

После открытия Reveal Region 5 (полное изображение) у игрока есть последняя попытка угадать фильм.

Если ответ снова неверный:

игра завершается поражением;
показывается сообщение «Увы, попробуйте в следующий раз»;
открывается правильный ответ.

14. Победа

После правильного ответа:

открывается всё изображение;
рассчитываются очки;
обновляется статистика;
обновляется серия;
показывается экран результатов.

14.1 Поражение

После неверной последней попытки (когда полное изображение уже открыто):

состояние игры — LOST;
Movie Score не начисляется;
показывается правильный ответ;
статистика обновляется (игры сыграно, победа не засчитывается, стрик сбрасывается).

15. Экран результатов

Показывается:

фильм;
итоговый Movie Score;
сколько Reveal Regions открыто;
время прохождения;
текущий стрик;
кнопка Share.

Percentile скрыт до появления достаточного количества прохождений.
В MVP экран результата отображается без процентиля.

16. Архив

Архив содержит все опубликованные игры.

Каждую можно:

пройти заново;
посмотреть результат;
открыть полностью.

Архивные игры не влияют на ежедневный рейтинг.

17. Persistence (MVP)

MVP сохраняет прогресс игрока только в localStorage.

Авторизация и синхронизация между устройствами переносятся на Version 1.1.

18. Edge Cases (очень важный раздел)

Вот этот раздел обычно забывают, а потом появляются баги.

Например:

Пользователь обновил страницу во время игры

→ Прогресс восстанавливается.

Пользователь сменил устройство

→ Прогресс синхронизируется после входа в аккаунт.

Пользователь угадал фильм с первой попытки

→ Начисляются максимальные очки.

Пользователь открыл полное изображение (Reveal Region 5), но не угадал с последней попытки

→ Поражение. Показывается ответ и сообщение «Увы, попробуйте в следующий раз».

Пользователь покинул игру

→ Состояние сохраняется автоматически.
