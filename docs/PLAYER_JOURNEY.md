# Player Journey

Продуктовый документ. Описывает путь игрока MovieDNA.

Любой новый PR должен улучшать этот путь.
Если PR не улучшает путь — сначала обсудить необходимость, не писать код.

Вопрос-фильтр: **поможет ли это игроку захотеть вернуться завтра?**

---

## Полный путь

```text
Открыл сайт
↓
Понял правила
↓
Начал Daily
↓
Получил первую эмоцию («ага!» / напряжение угадывания)
↓
Прошёл Challenge
↓
Увидел результат (Score, серия, следующий шаг)
↓
Перешёл в Архив
↓
Прошёл ещё один Challenge
↓
Вернулся на следующий день
↓
Получил Streak
↓
Поделился результатом
```

---

## Что должно быть правдой на каждом шаге

### Открыл сайт
Игрок сразу понимает: это ежедневная игра про угадывание фильма по картинке-ДНК, не по кадру из фильма.

### Понял правила
До кнопки «Начать» ясно: угадать фильм; открытие областей = подсказки и минус к Score; одна Daily в день.

### Начал Daily
Один главный Challenge сегодня. Архив не подменяет Daily.

### Первая эмоция
Честные подсказки, момент узнавания. Игрок не проигрывает из‑за непонятной загадки.

### Прошёл Challenge
Победа или поражение — оба исхода закрыты. Нет тупика.

### Увидел результат
Понятен Movie Score. Видна серия (после Daily). Есть действие дальше: Архив и/или «вернуться завтра».

### Перешёл в Архив
Архив — продолжение цикла («ещё один»), не свалка ссылок. Без спойлеров непройденных.

### Прошёл ещё один
Сыграл архивный Challenge → снова видит «что дальше», не выпадает из продукта.

### Вернулся на следующий день
Daily доступен. Серия растёт при победе в Daily. Архив серию не ломает и не заменяет.

### Получил Streak
В профиле и на экране результата серия читается как причина вернуться.

### Поделился результатом
Хочется скриншот / share друзьям. Share не ломает опыт, если недоступен.

## Roadmap по пути (кратко)

| PR | Шаг пути |
|----|----------|
| №1 Core Retention Loop | результат → архив / завтра / streak / профиль |
| №2 Эмоциональный результат | результат + share |
| №3 Архив | архив как вторая причина |
| №4 Первый опыт | правила до старта |
| №5 Навигация и края | пустые состояния, nav, скрыть мёртвые функции |

Контент (14–21 Challenge вперёд) — постоянный процесс вне нумерации PR.
