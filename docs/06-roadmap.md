# 06. Roadmap

## Принцип

Каждый следующий PR отвечает только на вопрос:

> Поможет ли это игроку захотеть вернуться завтра?

Если нет — задачу откладываем.

Engine, Content Studio и текущая архитектура достаточны для первого публичного релиза.
Приоритет — пользовательский опыт и обратная связь от игроков.

Контент (14–21 Challenge вперёд) — постоянный процесс, не пункт roadmap.

См. также: [PLAYER_JOURNEY.md](./PLAYER_JOURNEY.md).

## До публичного релиза

### PR №1 — Core Retention Loop

- Streak
- Profile
- Экран результата без тупика
- CTA в Архив
- CTA «вернуться завтра»
- Related / random после финиша (Daily и Archive)

### PR №2 — Эмоциональный экран результата

Movie Score, качество игры, регионы, время, желание поделиться.
Без изменения механики Score.

### PR №3 — Архив как вторая причина оставаться

Архив продолжает игровой цикл после Daily.

### PR №4 — Первый опыт нового игрока

Правила до старта, особенно mobile. Минимум текста.

### PR №5 — Навигация и крайние состояния

Пустой Daily, mobile nav, скрыть недоступный auth.
