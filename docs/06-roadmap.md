# 06. Roadmap

## Принцип

Каждый следующий PR отвечает только на вопрос:

> Поможет ли это игроку захотеть вернуться завтра?

Если нет — задачу откладываем.

Engine, Content Studio и текущая архитектура достаточны для первого публичного релиза.
Приоритет — пользовательский опыт и обратная связь от игроков.

Контент (14–21 Challenge вперёд) — постоянный процесс, не пункт roadmap.

См. также: [PLAYER_JOURNEY.md](./PLAYER_JOURNEY.md).

## Заморозка функциональности (после PR №5)

Не добавляем новые возможности «на всякий случай».

Следующий этап:

1. Заполнение контентом (14–21 Challenge вперёд)
2. Включение GA4
3. Первые реальные игроки
4. Сбор данных
5. Развитие только на основе поведения

Фильтр новой идеи: **поможет ли это получить первых игроков быстрее?**  
Если нет — backlog.

---

## Отложено (после soft launch / 100–500 игроков)

- Percentile / сравнение с игроками
- Auth и sync между устройствами
- Leaderboard
- Push / PWA
- Pinch-zoom
- Duplicate Level UI
- Фильтры в Архиве
- Локальная дата вместо UTC (если метрики покажут боль)
