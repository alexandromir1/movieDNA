## 1. Цель первого запуска

### Что должен почувствовать игрок (Expected Emotion — итог)

| Момент | Эмоция |
|--------|--------|
| 0–5 с | Интерес |
| 5–15 с | Любопытство / «узнаю?» |
| 15–30 с | Напряжение поиска → облегчение от фрагмента или лёгкое «мимо» |
| Момент узнавания | **«Аааа, понял!»** |
| После победы | **«Хочу ещё один.»** |

Не должен почувствовать: стыд, FOMO календаря, растерянность «куда нажать», скуку от инструкции.

### Что понять за первые 10 секунд

1. Нужно **угадать фильм**.  
2. Картинка открывается **фрагментами**.  
3. Можно **отвечать сразу**.  
4. Чем раньше — тем лучше.  
5. Сейчас — одна кнопка **Играть**.

### Какое действие захотеть после первого успешного фильма

> Сразу нажать **CTA продолжения** и начать второй фильм.

Не: день, архив, about, режим.

## 2. Путь по секундам

Идеальный сценарий: новый пользователь, фильм 1 лёгкий, успех в пределах ~минуты.  
Время этапов — ориентир сценария. Отдельно: **максимальное время без действия** и реакция UI.

---

### 0–5 секунд — «Что это?»

| | |
|--|--|
| **Состояние** | `NOT_STARTED` |
| **Expected Emotion** | **Интерес** |
| **Макс. бездействие** | **10 секунд** с момента показа экрана |

**Если ничего не нажал за 10 с**

- Допустимо: лёгкая пульсация / подсветка кнопки **Играть** (без текста-подсказки «нажмите сюда»).  
- Допустимо: ничего (если интерес уже держит взгляд на картинке).  
- Нельзя: модалка, туториал, автостарт игры без жеста, второй CTA.

**Что видит**

- MovieDNA  
- Три строки правил  
- **Играть**  
- Фон: нейтральный тизер **не из загадки фильма 1** (или без спойлера) — см. §10

**О чём думает:** «Про кино?», «Куда жать?»

**Действие:** смотрит → жмёт **Играть**

**Как направляем:** одна кнопка, короткий текст, без конкурентов

| Вопрос | Закрывает |
|--------|-----------|
| Что это? | Бренд + «Угадай фильм» |
| Что делать? | **Играть** |
| Читать долго? | 3 строки |

**UX Validation:** одна CTA? иначе упростить. Длинный текст → режет интерес.

**Проверка эмоции:** если за 5–10 с нет интереса / нет желания жать — экран мёртвый (перегрузить текстом или пустой).

---

### 5–15 секунд — «Я в игре»

| | |
|--|--|
| **Состояние** | `VIEWING_FRAGMENT` / `CONSIDERING` |
| **Expected Emotion** | **Любопытство** — «что это за фильм?» |
| **Макс. бездействие** | **20 секунд** на первом фрагменте без тапа |

**Если ничего не нажал за 20 с**

- Допустимо: едва заметная подсветка **одной** из кнопок (предпочтительно не обе сразу — иначе снова выбор «что важнее»). Рекомендация first-run: лёгкий акцент на **Следующий фрагмент**, если фрагмент совсем «молчаливый»; иначе только дыхание UI без давления.  
- Нельзя: авто-открытие фрагмента без жеста на L1 (ломает обучение «я сам нажал»).  
- Нельзя: всплывающий tutorial overlay.

**Что видит:** первый большой фрагмент + **Угадать фильм** + **Следующий фрагмент**

**О чём думает:** «Можно уже отвечать?», «Мало видно…»

**Действие:** смотрит → угадать или фрагмент

| Вопрос | Закрывает |
|--------|-----------|
| Можно отвечать сразу? | **Угадать фильм** |
| Что за кнопка? | **Следующий фрагмент** |

**Проверка эмоции:** если нет любопытства — слабый первый фрагмент (контент), не «добавить инструкцию».

---

### 15–30 секунд — «Ещё чуть-чуть / попытка»

| | |
|--|--|
| **Expected Emotion** | **Любопытство↑** при новом фрагменте; при ошибке — лёгкое «мимо», не стыд |
| **Макс. бездействие** | После ошибки / нового фрагмента снова ~**15–20 с**, затем та же мягкая подсветка, без модалки |

Ветки: фрагмент (A) / guess (B) / wrong (C) — все валидны.

**Что видит:** новый кусок и/или «неверно» + те же две кнопки

| Вопрос | Закрывает |
|--------|-----------|
| Если ошибусь? | «Неверно» + можно продолжать |
| Проиграл? | Нет |
| Кнопка сработала? | Явный новый фрагмент |

**На L1:** фрагмент открывается **только по кнопке** (не авто при ошибке) — предсказуемее для первого жеста. (Связь с Q1 в game-loop.)

**Проверка эмоции:** злость / «всё сломано» после ошибки = провал UI ошибки.

---

### 30–60 секунд — «Ага!» → результат → продолжение

| | |
|--|--|
| **Состояния** | `WON` → `RESULT` → следующий фильм |
| **Expected Emotion** | **«Аааа, понял!»** → гордость → **«Хочу ещё один.»** |
| **Макс. бездействие на результате** | **8–12 секунд** |

**Если на результате ничего не нажал за ~10 с**

- Допустимо: подсветить **CTA продолжения**.  
- Нельзя: автопереход без возможности увидеть название/эмоцию победы (см. §5 «Что нельзя потерять»).  
- Нельзя: заменить primary на share.

**Что видит**

1. Успех + название  
2. Короткий результат  
3. Одна **CTA продолжения** (лейбл — TODO)  
4. Вторичное (share и т.д.) — слабее, не обязательно в первую минуту  

**Действие:** жмёт CTA продолжения → сразу первый фрагмент фильма 2

| Вопрос | Закрывает |
|--------|-----------|
| Угадал? | Название + win |
| Что дальше? | **CTA продолжения** |
| Ждать завтра? | Такого UI нет |

**Проверка эмоции:** нет «хочу ещё» → виноваты конкурирующие кнопки или скучный/долгий результат, не «мало текста».

## 3. Что чувствует игрок (сводка Expected Emotion)

| Этап | Expected Emotion | Если эмоции нет — подозревать |
|------|------------------|-------------------------------|
| 0–5 с | Интерес | Перегруз / пустой экран / много CTA |
| 5–15 с | Любопытство | Слабый фрагмент, жаргон на кнопках |
| 15–30 с | Любопытство / азарт | Невидимый новый фрагмент; жёсткий fail |
| Узнавание | **«Аааа, понял!»** | Слишком рано спойлер или слишком поздно пустота |
| После победы | **«Хочу ещё один.»** | Архив/share/опрос перебили импульс |

Эмоция — критерий приёмки UX наравне с «понял ли кнопку».

## 5. Что нельзя потерять

Критичные моменты first-run. Ломаются на тестах чаще, чем «красивый макет».

### После угадывания фильма

Нельзя:

- ❌ Закрыть / сменить экран **раньше** короткого поздравления и показа названия  
- ❌ Случайным **«назад»** браузера / жестом уйти в пустоту без восстановления результата или прогресса  
- ❌ **Пропустить результат** автопереходом настолько быстрым, что игрок не успел увидеть название и эмоцию  
- ❌ Показать результат без **CTA продолжения** (тупик «что дальше?»)  
- ❌ Подменить primary на share / подборку / опрос  

Можно:

- Короткий win-beat (доли секунды–пара секунд) → результат с названием  
- Затем игрок сам жмёт CTA продолжения (предпочтительно на first-run)  
- Мягкая подсветка CTA при бездействии ~10 с — не автостарт следующего фильма до просмотра победы  

### На главном (0–5 с)

Нельзя потерять: единственность **Играть**; три строки смысла.

### На уровне

Нельзя потерять: возможность угадать **сразу**; явный эффект «Следующий фрагмент».

### При уходе mid-level

Нельзя потерять: прогресс очереди (и желательно открытые фрагменты) — см. user-flow `AWAY`.

## 6. Первый успешный опыт (момент узнавания)

**Что это:** секунда, когда название складывается в голове по фрагментам.

**Почему удовольствие:** сам собрал смысл; напряжение неполной картины; L1 лёгкий; без календаря и жаргона.

**Почему «хочу ещё»:** паттерн награждён; CTA продолжения ловит импульс сразу; архив/день импульс убивают.

**Контент L1:** сильные символы, честный первый фрагмент.

---

## 7. Первый цикл удержания

Цель: результат → сразу фильм 2.

### Подталкивает

| Элемент | Зачем |
|---------|--------|
| Одна **CTA продолжения** | Одно действие |
| Выше fold | Не искать |
| Короткий результат | Не читать вместо «ещё» |
| Мгновенный старт фильма 2 | Паттерн уже знаком |
| Лёгкий L1 | Победа свежая |

### Мешает

Share/архив/дом как primary; опрос; киномарафон-primary; открытый breakdown; хаб выбора; «приходи завтра».

### Нельзя

Календарь, Daily, режим, туториал перед #2, две primary, Challenge/Reveal/регион.

## 8. Аналитика (first-run)

Всё через `analytics.track()`.  
Для каждого события: **что измеряем** и **зачем через полгода** (чтобы не забыть смысл).

### Must-have цепочка

```text
home_open
  → start_button_clicked
  → level_started (1)
  → … (фрагменты / догадки)
  → level_completed
  → next_level_clicked    ← технический id; UI-лейбл CTA продолжения — TODO
  → level_started (2)
```

| Событие | Когда | Зачем измеряем (цель метрики) |
|---------|--------|-------------------------------|
| `home_open` | Открыл главный | Знаем размер входа в продукт; база воронки first-run |
| `start_button_clicked` | Нажал **Играть** | Конверсия «увидел → начал»; если низкая — проблема главного экрана/интереса 0–5 с |
| `level_started` | Старт фильма N | Сколько реально вошли в игру; `levelIndex` отличает 1-й и 2-й фильм |
| `fragment_revealed` | Жмёт **Следующий фрагмент** | **Скорость/частота открытия подсказок**; рано ли «сдаются» фрагментам vs угадывают вслепую; баланс L1 |
| `guess_submitted` | Отправил ответ | **Насколько рано начинают отвечать**; попытки до успеха |
| `search_used` | Выбрал из списка | Работает ли поиск в первый раз; не ломает ли first-run ввод |
| `wrong_guess` | Ошибка | Трение L1; не слишком ли жёсткий первый фильм |
| `correct_guess` | Верно | Факт узнавания в сессии |
| `level_completed` | Фильм пройден | Доля завершивших L1; связка с числом фрагментов/попыток |
| `next_level_clicked` | Жмёт CTA продолжения | **Главный индикатор удержания first-run**: сработала ли эмоция «хочу ещё» |
| `level_started` (2) | Начал второй | Success Criteria: сам пошёл дальше без обучения |
| `level_abandoned` | Ушёл до конца L1 | Обрыв в первые минуты; где теряем (до/после первого фрагмента) |

### Опционально (не блокируют первый PR)

| Событие | Зачем |
|---------|--------|
| `first_run_viewed` | Отделить чистых новых от `home_open` вернувшихся |
| `guess_ui_opened` | Угадывают ли до любого фрагмента |
| `result_viewed` | Дошли до результата, но не нажали CTA продолжения |

Daily/archive-события **не** success-метрики first-run.

## 9. UX Validation

| Этап | Без объяснений? | Ошибка из‑за UI? | Думает вместо игры? | Нет Expected Emotion? |
|------|-----------------|------------------|---------------------|------------------------|
| 0–5 | Да | >1 CTA | Стена текста | Нет интереса → упростить |
| 5–15 | Да | Жаргон | 3-я кнопка | Нет любопытства → контент фрагмента |
| 15–30 | Да | Fail = «конец» | Неясный фрагмент | Злость → смягчить fail |
| 30–60 | Да | Primary не next | Опрос/архив | Нет «ещё» → убрать конкурентов |

Фильтр: нужен объяснитель рядом → этап провален.

## 10. First Run Success Criteria

Успех, если новый пользователь:

1. Понял правила без туториала.  
2. Сам прошёл первый фильм.  
3. Сам начал второй (CTA продолжения).  
4. Не спрашивал «что делать дальше?».  
5. Не видел неоднозначных primary.  
6. Не видел календарь / режим / reset.  
7. Было **«Аааа, понял!»** и импульс **«хочу ещё»**.  
8. В аналитике есть цепочка до `level_started`(2).  
9. Не потерял момент победы (название + эмоция до ухода дальше).

Провал: «куда дальше?», массовый уход в share, жёсткий L1 без «ага!».

## 11. Максимальное время без действия (сводка)

| Экран / этап | Макс. idle | Реакция UI | Запрещено |
|--------------|------------|------------|-----------|
| Главный | 10 с | Подсветка **Играть** или ничего | Автостарт, модалка, 2-я CTA |
| Первый фрагмент | 20 с | Мягкий акцент (осторожно) | Авто-фрагмент на L1, tutorial |
| После ошибки / фрагмента | 15–20 с | Та же мягкость | Смена правил |
| Результат | 8–12 с | Подсветка **CTA продолжения** | Автопереход до просмотра победы; подмена primary |

Idle-подсказки **не заменяют** ясность первой отрисовки.

## 14. Формула первых 60 секунд

> Интерес → **Играть** → любопытство к фрагменту → **«Аааа, понял!»** → победа → **«Хочу ещё один.»** → **CTA продолжения** → второй фильм.  
> Без календаря. Без туториала. Без вопроса «что делать дальше?»  
> Название кнопки продолжения — **TODO после UX-аудита терминологии**.
