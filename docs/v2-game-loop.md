## 1. Причина изменений

После глубинных интервью:

| Сигнал | Проблема v1 |
|--------|-------------|
| Много вопросов о правилах | Daily + регионы + Score + архив + даты |
| «Что открыть?» | Календарь заставляет выбирать |
| Daily = лишняя нагрузка | «Нужно успеть сегодня» вместо «хочу ещё фильм» |
| Хотят продолжать играть | После результата развилки (вчера / архив / завтра) |
| Просят контент, не календарь | Уровни спрятаны за датами |

**Вывод:** ежедневная викторина мешает циклу «узнал → угадал → ещё».

---

## 2. Философия продукта v2

MovieDNA **не** ежедневная викторина.

MovieDNA — **игра с линейной прогрессией**.

### 2.1. Один принцип UX

Игрок всегда знает только одно действие:

> **Продолжить игру.**

На всём главном пути:

- нет выбора уровней;
- нет дат и календаря;
- нет архива как входа в игру;
- нет выбора «режима» раскрытия;
- нет Daily / «сегодняшнего фильма».

### 2.2. Что остаётся из v1

1. Момент «Ага!» — ядро.
2. Честные подсказки.
3. Художественная ДНК фильма (не кадр / не постер / не лица).
4. Короткая сессия (ориентир 1–3 минуты на уровень).
5. Чем раньше угадал — тем лучше результат.

### 2.3. Было → стало

| v1 | v2 |
|----|----|
| Daily Challenge по дате | Линейная очередь уровней |
| Архив как вторая сессия | Тот же прогресс: следующий уровень |
| «Что открыть сегодня?» | «Играть» / «Продолжить» |
| Reveal Regions | Фрагменты (и позже другие стратегии) |
| Календарь = прогресс | Позиция в очереди = прогресс |
| Слова Challenge / Reveal / регион в UI | Человеческий язык: уровень, фрагмент, угадать |

## 3. Игровой цикл

```text
Главный экран
    │
    ▼
Играть  (новый)  /  Продолжить  (есть прогресс)
    │
    ▼
Уровень N   ← всегда следующий по очереди
    │
    ▼
Угадал
    │
    ▼
Короткий результат
    │
    ▼
Уровень N+1   ← без выбора
```

Пример:

```text
Уровень 17 → победа → Уровень 18
```

### 3.1. Правила

1. После победы (завершения уровня) игрок переходит к **следующему** уровню.
2. Переход не требует выбора «какой уровень».
3. Главный экран без сетки, списка и календаря.
4. CTA: **Играть** (нет прогресса) или **Продолжить** (есть прогресс; можно подписать «Уровень N»).
5. Прогресс **сохраняется между сессиями**.
6. Игрок **никогда не начинает сначала**: возврат всегда на текущую позицию очереди. Нет «сбросить прогресс» / «начать с уровня 1» на главном пути и в базовом продукте v2.0.

### 3.2. Что игрок не решает

- какой день открыть;
- daily vs архив;
- какой режим раскрытия;
- с какого уровня начать заново.

## 4. Уровни и прогрессия

### 4.1. Термины

| Для игрока | В системе | Запрещено в UI |
|------------|-----------|----------------|
| Уровень N | элемент `LevelSequence` | Challenge |
| Фрагмент | шаг Reveal Strategy `fragments` | регион, Reveal Region |
| Угадать фильм | guess flow | — |
| Следующий фрагмент | `revealNext()` | «открыть подсказку» как жаргон Engine |

Слова **Challenge**, **Reveal**, **регион** — только во внутренней документации/коде до полной зачистки, **не в интерфейсе**.

### 4.2. Очередь

```text
Уровень 1 → 2 → 3 → … → N → N+1
```

- Всегда выдаётся следующий непройденный уровень.
- Повтор пройденных **не входит** в v2.0 основной цикл (см. открытые вопросы).
- Новый контент = уровни в конец очереди, не слоты дат.

### 4.3. Кривая сложности

Задаётся автором заранее, например:

```text
лёгкий → лёгкий → лёгкий → средний → лёгкий → средний → тяжёлый → …
```

Цели: обучить без туториала → уверенность → пик эмоции → выдох.

### 4.4. Прогресс игрока (концепт)

| Поле | Назначение |
|------|------------|
| `currentLevelIndex` | Куда ведёт «Продолжить» |
| `completedLevelIds[]` | Пройденные уровни |
| `bestScoreByLevel` | Опционально; для будущего повтора |
| `unlockedRevealMethods[]` | Для будущего unlock; в v2.0 всегда только `fragments` |

Хранение: сначала локально; позже аккаунт. Прогрессия **не зависит** от auth.

### 4.5. Retention

Фильтр PR:

> Помогает ли это быстрее дойти до следующего уровня и захотеть продолжить?

Daily streak **не** ядро v2 и не ограничивает доступ к уровням.

## 5. Механика: Фрагменты

### 5.1. Единственная механика релиза

В v2.0 существует только **Фрагменты**.

Название должно быть понятно новому игроку: слово «фрагмент» используется в кнопке и коротком описании правил. Отдельного экрана «выберите режим: Фрагменты» **нет**.

Другие механики **не реализовывать**. В документации — только архитектурная возможность (§8–9).

### 5.2. Как выглядит

1. Виден **только первый большой фрагмент**.
2. Остальной холст **отсутствует** — не затемнён, не в квадратах, не blur. Его нет.
3. **🧩 Следующий фрагмент** — появляется следующий крупный кусок.
4. Затем третий, четвёртый…
5. Картинка **собирается** крупными частями.

**Не является:** пазлом, drag-and-drop, сеткой тайлов, Reveal Regions v1 (маска поверх полной картины).

### 5.3. Две кнопки (обязательный UI)

| Кнопка | Действие |
|--------|----------|
| 🎬 **Угадать фильм** | Ввод / поиск названия |
| 🧩 **Следующий фрагмент** | Добавляет следующий фрагмент (влияет на результат) |

Вторичное (язык, закрыть поиск) — не конкурирует с этими двумя.

### 5.4. Правила шагов

1. Порядок фрагментов задаёт контент; игрок не выбирает кусок.
2. Когда фрагменты закончились — кнопка «Следующий фрагмент» недоступна / «Все фрагменты открыты»; угадывание возможно.
3. Поведение при неверном ответе — см. открытый вопрос **Q1** (не блокирует старт архитектуры).

### 5.5. Контент

Автор задаёт:

- полное изображение;
- упорядоченный список фрагментов (формат геометрии — на этапе Studio/PR);
- число фрагментов (ориентир 4–5; точное правило — в `03-economy` / content после Q5).

Каждый следующий фрагмент заметно повышает шанс узнать фильм.

## 6. Интерфейс и обучение

### 6.1. Главный / первый экран

Убрать лишнее. Понять правила за несколько секунд.

Короткое описание (утверждённый текст):

> Угадай фильм.  
> Не уверен? Открывай новые фрагменты.  
> Чем раньше угадаешь — тем лучше результат.

Одна CTA: **Играть** или **Продолжить**.

Без календаря, архива, выбора сложности, выбора режима.

### 6.2. Экран уровня

Только игровое поле + две кнопки из §5.3. Без жаргона Engine.

### 6.3. Экран результата

1. Эмоция + результат уровня.  
2. Название фильма.  
3. Главная **CTA продолжения** (или автопереход с той же семантикой).  
   > **TODO:** финальный лейбл кнопки — после UX-аудита терминологии (не фиксируем «Следующий уровень» / «Следующий фильм»).  
4. Вторично: share, киномарафон, детали счёта.

Киномарафон не заменяет CTA продолжения.

### 6.4. Анти-принципы UI

- Не показывать Blur / Focus / Color / Light.
- Не спрашивать Daily / архив.
- Не показывать список уровней для старта игры.
- Не использовать Challenge / Reveal / регион в копирайте.

## 8. Будущие способы раскрытия (только архитектура)

| Способ | Идея (не в UI v2.0) |
|--------|---------------------|
| Фрагменты | Сборка крупными частями — **единственный способ релиза** |
| Размытие | Из blur |
| Фокус | Проявление резкости |
| Цвет | Монохром → цвет |
| Свет | Экспозиция / проявка |

Правила:

1. Игроку **не показывать** и **не давать выбор режима**.
2. После запуска все начинают во **Фрагментах**.
3. Другие способы позже открываются постепенно по прогрессу.
4. Добавление способа не ломает цикл «Продолжить → уровень → следующий».
5. Один уровень = один способ раскрытия.

## 9. Архитектура Engine

Игровая логика **не зависит** от конкретной стратегии открытия изображения.

### 9.1. Слои

```text
Presentation (UI)
    │  «Угадать фильм» / «Следующий фрагмент»
    ▼
Game Engine / Session
    │  очередь уровней, попытки, score hooks, next level
    ▼
Reveal Strategy   ← интерфейс
    │
    ├── Fragments     ← единственная реализация v2.0
    ├── Blur          ← позже
    ├── Focus         ← позже
    ├── Color         ← позже
    └── Light         ← позже
    │
    ▼
Reveal Renderer
```

### 9.2. Контракт Reveal Strategy

Стратегия:

| Понятие | Назначение |
|---------|------------|
| `totalSteps` | Число шагов |
| `openedSteps` | Уже открыто |
| `canRevealNext` | Есть ли ещё шаг |
| `revealNext()` | Открыть следующий |
| `isFullyRevealed` | Всё открыто |
| `getViewState()` | Данные для Renderer |

Стратегия **не** делает: валидацию ответа, поиск, сохранение прогресса очереди, навигацию, PostHog.

### 9.3. Game Engine

- старт уровня из прогрессии;
- цикл угадывания;
- `revealNext` по кнопке (и опционально по ошибке — Q1);
- результат уровня;
- переход к `N+1`;
- события аналитики через `analytics.track`.

### 9.4. Legacy Reveal Regions

v1 Regions — не продуктовая модель v2. На миграции: конвертация в Fragments или временный internal adapter. Игроку не показывать.

## 10. Модель контента

| Сущность | Роль v2 |
|----------|---------|
| **Movie** | Названия, год, aliases, рекомендации |
| **Level** | image, difficulty, answers, `revealMethod: fragments`, данные фрагментов |

Публикация = добавить уровень в sequence.

Календарь scheduling — снят с главного пайплайна.

## 11. Экономика (рамка)

Инварианты v2:

1. Раньше угадал (меньше фрагментов) → лучше результат.
2. «Следующий фрагмент» предсказуемо ухудшает потенциальный результат.
3. Нет энергии и paywall на «Продолжить».
4. Streak не управляет доступом к уровням.

Числа → обновление `03-economy.md` отдельным шагом (см. миграцию docs).

## 12. Аналитика (рамка)

Только `analytics.track()`.

Целевая воронка v2:

```text
home_open
  → start_button_clicked | continue_clicked
  → level_started
  → fragment_revealed
  → guess_* 
  → level_completed
  → next_level (auto или click)
```

`home_open` / daily / archive как core funnel — устаревают.

## 17. Статус: архитектура завершена → реализация

Архитектура v2 **завершена** (вместе с `v2-user-flow`, `v2-first-run`, `v2-domain-model` + Engine Boundaries).

Дальше — только **небольшие PR**. Новые архитектурные документы — лишь при реальном пробеле, который нельзя закрыть правкой существующих SoT.

### 17.1. Процесс разработки

```text
Issue
  → один небольшой PR
  → тестирование
  → интервью пользователей
  → следующий PR
```

- Не предлагать большие рефакторинги без явной причины.  
- Не менять архитектуру без пользовательских данных (интервью / аналитика).  
- Идея улучшения → проверить: какую боль игрока решает; есть ли подтверждение; можно ли без ломки фундамента. Иначе — отложить.

После каждого PR в описании: что сделано · зачем · влияние на дальше · соответствие SoT.

### 17.2. План первых PR

| PR | Содержание | Вне scope |
|----|------------|-----------|
| **1** | Игровой цикл v2 за flag/`/v2`: без Daily на этом пути; **LevelSequence**; **Player Progress**; **Continue** с места остановки; production v1 не трогать | Новые механики раскрытия |
| **2** | Home: без календаря/архива/дат; простой first screen (смысл за секунды) | — |
| **3** | Игровой экран: только **Угадать фильм** / **Следующий фрагмент**; язык без Challenge/Reveal/Region/Daily | Blur и др. |
| **4** | Результат: мотивация к следующему Level; secondary (share / киномарафон) не перебивают CTA продолжения | Новые механики |
| **5** | Стойкое сохранение прогресса (всегда продолжить с того же места) | Sync-аккаунт, если ещё нет |

### 17.3. Пока не реализовывать

### 17.4. Definition of Ready → PR1

2. ✅ План PR1–6 принят.  
3. Q1–Q2 не блокируют PR1 (цикл + progress); уточняются к PR3–4.
