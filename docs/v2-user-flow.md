# MovieDNA v2 — User Flow

**Статус:** продуктовый документ. Путь игрока, состояния, цели экранов, запреты, ошибки.  
**Источник истины по концепции:** [v2-game-loop.md](./v2-game-loop.md).  
**Код не писать** только по этому файлу — сначала согласование UX.

Язык — простой, для продукта и дизайна. Без Engine API и JSON.  
Состояния из §2 потом лягут в Engine Session.

---

## 1. Общая философия

### Главный принцип

> Игрок никогда не должен думать: «Что делать дальше?»

В каждый момент интерфейс предлагает **одно очевидное следующее действие**.

### Как это выглядит на практике

| Момент | Одно действие |
|--------|----------------|
| Первый заход | **Играть** |
| Вернулся | **Продолжить** |
| Смотрит картинку | **Угадать фильм** или **Следующий фрагмент** |
| Угадал | **CTA продолжения** (лейбл — TODO после UX-аудита терминологии) |
| Через несколько дней | Снова **Продолжить** — с того же места |

**Почему так:** любое второе «главное» действие возвращает вопрос «что открыть?» — именно его мы убирали, отказываясь от Daily и календаря.

Нет календаря. Нет «какой день открыть». Нет архива на входе. Нет выбора режима.

## 2. User State Machine

Состояния игрока — не экраны. Экран показывает состояние; Engine потом хранит переходы.

### 2.1. Основная цепочка (счастливый путь)

```text
NOT_STARTED          Не начал
    │
    ▼
VIEWING_FRAGMENT     Смотрит фрагмент(ы)
    │
    ▼
CONSIDERING          Думает (всё ещё на том же экране)
    │
    ├──► GUESSING    Пишет ответ
    │       │
    │       ├── wrong ──► VIEWING_FRAGMENT   (можно снова думать / открыть фрагмент)
    │       │
    │       └── correct ──► WON
    │                          │
    │                          ▼
    │                       RESULT           Результат
    │                          │
    │                          ▼
    │                       NEXT_LEVEL       Переход → снова VIEWING_FRAGMENT
    │
    └──► REVEALING_NEXT   Нажал «Следующий фрагмент»
            │
            ▼
         VIEWING_FRAGMENT   (фрагментов стало больше)
```

### 2.2. Таблица состояний

| Состояние | Что делает игрок | Что видит | Следующее очевидное действие |
|-----------|------------------|-----------|------------------------------|
| `NOT_STARTED` | Ещё не играл / на главном без активной сессии | **Играть** или **Продолжить** | Начать |
| `VIEWING_FRAGMENT` | Смотрит картинку | Фрагменты + 2 кнопки | Угадать или открыть фрагмент |
| `CONSIDERING` | Думает (подсостояние просмотра) | То же | То же — без нового экрана |
| `GUESSING` | Пишет / выбирает название | Поле поиска | Отправить ответ или закрыть ввод |
| `REVEALING_NEXT` | Короткий миг открытия фрагмента | Появление куска | Снова смотреть / угадывать |
| `WRONG_GUESS` | Только что ошибся | «Неверно» + те же 2 кнопки | Снова угадать или фрагмент |
| `WON` | Верный ответ, миг победы | Успех | Авто → результат |
| `RESULT` | Смотрит итог уровня | Счёт + название + **CTA продолжения** | Следующий фильм |
| `BETWEEN_LEVELS` | Переход | Почти незаметен | Сразу новый фрагмент |
| `AWAY` | Закрыл вкладку / ушёл | — | При возврате → `NOT_STARTED` с прогрессом → **Продолжить** |
| `STUCK` | Не знает ответ, фрагменты ещё есть | 2 кнопки | Открыть фрагмент |
| `ALL_FRAGMENTS_OPEN` | Всё открыто, ещё не угадал | Угадывание; кнопка фрагмента недоступна | Угадать (или сдача — если разрешим позже) |

`CONSIDERING` **не отдельный экран** — это то же `VIEWING_FRAGMENT`. Выделено, чтобы Engine и аналитика отличали «смотрит» от «уже печатает».

### 2.3. Переходы (для разработки)

| Из | Событие | В |
|----|---------|---|
| `NOT_STARTED` | Играть / Продолжить | `VIEWING_FRAGMENT` |
| `VIEWING_FRAGMENT` | Угадать фильм | `GUESSING` |
| `VIEWING_FRAGMENT` | Следующий фрагмент | `REVEALING_NEXT` → `VIEWING_FRAGMENT` |
| `GUESSING` | Отмена / закрыл ввод | `VIEWING_FRAGMENT` |
| `GUESSING` | Неверный ответ | `WRONG_GUESS` → `VIEWING_FRAGMENT` |
| `GUESSING` | Верный ответ | `WON` → `RESULT` |
| `RESULT` | CTA продолжения | `BETWEEN_LEVELS` → `VIEWING_FRAGMENT` |
| Любое игровое | Закрыл вкладку | `AWAY` (прогресс уровня/очереди сохранён) |
| `AWAY` | Открыл сайт снова | `NOT_STARTED` (с CTA **Продолжить**) |

**Почему state machine отдельно от экранов:** экранов мало, состояний больше. Иначе в коде снова появятся флаги «показали ли угадалку», «уже выиграли», без единой картины.

## 3. Полный путь игрока (экран за экраном)

```text
Первый запуск
  → Главный экран
  → Первый фильм
  → (ошибка / фрагмент — по желанию)
  → Правильный ответ
  → Результат
  → Следующий фильм
  → … 
  → Возврат через дни → Продолжить
```

У каждого экрана: **цель**, **почему**, что видит, вопросы, обязательное, UX-проверка.

---

### 3.1. Первый запуск

**Цель:** за несколько секунд понять «это угадайка фильма по кускам картинки» и нажать **Играть**.

**Почему этот экран существует:** без него человек не знает жанр продукта.  
**Почему он короткий:** длинный текст = люди не читают и уходят; правила должны жить в кнопках на следующем шаге.

**Состояние:** `NOT_STARTED`

**Что видит**

- MovieDNA  
- Три строки:  
  > Угадай фильм.  
  > Не уверен? Открывай новые фрагменты.  
  > Чем раньше угадаешь — тем лучше результат.  
- **Играть**  
- Можно атмосферу — первый фрагмент как фон, без сетки уровней

| Вопрос игрока | Ответ интерфейса |
|---------------|------------------|
| Что это? | «Угадай фильм» + картинка |
| Что делать? | Одна кнопка **Играть** |
| Это на каждый день? | Календаря нет — вопрос отпадает |
| С чего начать? | Выбора нет |

**Обязательно:** бренд, 3 строки, **Играть**  
**Убрать:** архив, даты, streak, режимы, туториал-wizard

**UX-проверка:** одна кнопка? → да. Иначе упростить.

---

### 3.2. Главный экран

**Цель:** максимально быстро начать или продолжить игру.

**Почему:** любой лишний блок здесь конкурирует с «продолжить» и возвращает когнитивную нагрузку Daily.

**Состояние:** `NOT_STARTED` (с прогрессом или без)

| Состояние прогресса | CTA |
|---------------------|-----|
| Пусто | **Играть** |
| Есть | **Продолжить** (+ можно «Уровень N») |

| Вопрос | Ответ |
|--------|--------|
| Где я? | Продолжить / Уровень N |
| Что сегодня? | Вопроса нет |
| Другой уровень? | Не на этом экране |

**Обязательно:** одна CTA  
**Убрать:** календарь, архив равного веса, «вчера»

**UX-проверка:** если рядом две равные кнопки — одна лишняя.

---

### 3.3. Экран уровня (первый и любой следующий фильм)

**Цель:** дать момент «ага!» и принять одно из двух решений — угадать сейчас или открыть ещё фрагмент.

**Почему две кнопки, а не одна:** это единственное осмысленное решение в игре (риск vs информация).  
**Почему не три:** третья («сдаться», «меню», «режим») снова создаёт «что нажать?».  
**Почему картинка неполная:** пустой холст = честно «ещё не всё видно», без фальшивой «затемнённой полной сцены».

**Состояния:** `VIEWING_FRAGMENT` → `CONSIDERING` → `GUESSING` / `REVEALING_NEXT`

**Что видит:** фрагмент(ы) + **Угадать фильм** + **Следующий фрагмент**

| Вопрос | Ответ |
|--------|--------|
| Можно отвечать сейчас? | Да |
| Мало видно? | Следующий фрагмент |
| Это пазл? | Нет, куски сами появляются |
| Раньше = лучше? | Да (из первого экрана) |

**Обязательно:** картинка-фрагменты, две кнопки  
**Убрать:** Reveal, регион, Challenge, таймер-давление, выбор режима

**UX-проверка:** подписи человеческие? иначе переименовать.

---

### 3.4. Неправильный ответ

**Цель:** не наказать ощущением «конец», а вернуть в цикл угадай / фрагмент.

**Почему без отдельного «экрана поражения» на каждую ошибку:** ошибка — частый шаг обучения, не финал уровня.

**Состояние:** `WRONG_GUESS` → `VIEWING_FRAGMENT`

| Вопрос | Ответ |
|--------|--------|
| Я проиграл? | Нет |
| Что дальше? | Те же две кнопки |
| Штраф? | Счёт — на результате; сейчас не читать лекцию |

**Обязательно:** короткое «неверно» + возврат к двум действиям

**UX-проверка:** не добавлять третью главную кнопку в момент ошибки.

---

### 3.5. Открытие следующего фрагмента

**Цель:** заметно повысить шанс узнать фильм и усилить «ещё чуть-чуть».

**Почему по кнопке, а не всё сразу:** ценность раннего угадывания.  
**Почему по порядку, без выбора куска:** выбор куска = мини-головоломка поверх игры.

**Состояние:** `REVEALING_NEXT` → `VIEWING_FRAGMENT`

| Вопрос | Ответ |
|--------|--------|
| Что изменилось? | Новый крупный кусок |
| Обязательно? | Нет |
| Когда конец фрагментов? | Кнопка гаснет / «все открыты» |

**Эмоция:** «понятнее» → иногда сразу **«Я понял!»**

---

### 3.6. Правильный ответ (миг победы)

**Цель:** зафиксировать эмоцию узнавания, не развилку меню.

**Почему почти без UI:** пик **«Я понял!»** легко убить лишними кнопками.

**Состояние:** `WON` → `RESULT`

**Точка выбора:** нет.

---

### 3.7. Экран результата

**Цель:** вызвать желание пройти следующий фильм.

**Почему не «дом / архив / завтра»:** после победы мозг готов к ещё одному — это главный retention v2.  
**Почему счёт здесь, а не во время ошибки:** награда в конце, не наказание в процессе.

**Состояние:** `RESULT`

**Порядок:**

1. Эмоция + результат  
2. Название фильма  
3. **CTA продолжения** (главное; **TODO:** лейбл после UX-аудита терминологии)  
4. Слабее: share, киномарафон, детали  

| Вопрос | Ответ |
|--------|--------|
| Как сыграл? | Крупно сверху |
| Что дальше? | Одна CTA продолжения |
| Ждать завтра? | Нет |

**UX-проверка:** primary одна и выше fold.

---

### 3.8. Переход к следующему фильму

**Цель:** без трения начать новый цикл угадывания.

**Почему нет выбора между фильмами:** решение уже принято продуктом (очередь). Игрок не должен выбирать — только играть.

**Состояние:** `BETWEEN_LEVELS` → `VIEWING_FRAGMENT`

**UX-проверка:** нет хаба «выберите уровень 2».

---

### 3.9. Возврат через несколько дней

**Цель:** снять тревогу «всё пропало / пропустил дни» и сразу дать **Продолжить**.

**Почему сохраняем позицию, а не «фильм дня»:** обещание v2 — никогда не начинать сначала и не жить календарём.

**Состояние:** `AWAY` → `NOT_STARTED` (с прогрессом)

| Вопрос | Ответ |
|--------|--------|
| Потерял прогресс? | Нет |
| Штраф за дни? | Нет |
| С чего? | С того же места |

---

## 4. Что запрещено

Жёсткие правила. Нарушение = регресс к проблемам интервью.

### На любом экране

- ❌ Больше **одной** основной кнопки действия (на уровне исключение: ровно две игровые — угадать / фрагмент; они не «две главные CTA навигации», а одно решение из двух)
- ❌ На главном и результате — две равноправные primary
- ❌ Обязательный туториал / wizard из нескольких шагов
- ❌ Длинные инструкции «прочитайте перед игрой»
- ❌ Внутренние термины в UI: Challenge, Reveal, регион, Daily, strategy, mode
- ❌ Календарь, даты, «сегодняшний фильм» на пути игры
- ❌ Выбор уровня / режима / «с какого начать» на главном пути
- ❌ Сброс прогресса / «начать сначала» как заметное действие
- ❌ Экран между результатом и следующим фильмом с выбором
- ❌ Наказание за пропуск дней (FOMO-копирайт)

### На экране уровня

- ❌ Выбор «какой фрагмент открыть»
- ❌ Перетаскивание пазла
- ❌ Третья главная кнопка рядом с двумя игровыми

### На результате

- ❌ Primary «В архив» / «Домой» рядом с CTA продолжения
- ❌ Обязательный опрос до продолжения

**Почему список запретов:** через год «маленькая кнопка архива» снова создаст вопрос «что открыть?» — мы уже знаем, что это ломает продукт.

## 5. Где игрок принимает решение

| # | Где | Выбор | Почему допустим |
|---|-----|--------|-----------------|
| 1 | Главный | Играть/Продолжить или уйти | Вход в продукт |
| 2 | Уровень | Угадать сейчас **или** открыть фрагмент | Единственное игровое напряжение |
| 3 | Ввод | Какое название | Суть угадайки |
| 4 | Результат | Дальше (главное); share/подборка вторично | Не блокирует прогресс |

**Не решения v2:** день, архив, режим, номер уровня в очереди.

## 6. Эмоциональный подъём

| Момент | Внутри | Как не убить |
|--------|--------|--------------|
| Узнал по фрагменту | **«Я понял!»** | Не перекрывать картинку |
| Верный ввод | «Блин, точно!» | Короткий win-beat |
| Результат → CTA | **«Ещё один фильм.»** | Одна яркая кнопка |
| Ещё фрагмент / ещё попытка | **«Попробую ещё.»** | Сразу доступно угадывание |
| Возврат через дни | Спокойствие | Без «вы пропустили Daily» |

## 7. Ошибки и крайние случаи

Что происходит, когда путь ломается.

| Ситуация | Состояние | Что делает продукт | Чего не делать |
|----------|-----------|--------------------|----------------|
| Ввёл неправильный фильм | `WRONG_GUESS` | Коротко «неверно» → снова фрагменты и 2 кнопки | Экран «вы проиграли уровень» на каждую ошибку |
| Не знает ответ | `STUCK` | Может жать **Следующий фрагмент**; угадывание всегда доступно | Обязать читать подсказку-текст |
| Все фрагменты открыты, не угадал | `ALL_FRAGMENTS_OPEN` | Кнопка фрагмента недоступна; остаётся угадывание | Тупик без действий (нужен ясный путь; сдача — отдельное решение Q2) |
| Закрыл вкладку mid-level | `AWAY` | Сохранить прогресс очереди; прогресс фрагментов уровня — желательно сохранить | Сбросить на уровень 1 |
| Вернулся через неделю | `NOT_STARTED` + прогресс | **Продолжить** с того же места | «Начните сегодняшний Challenge» |
| «Проиграл» (если сдался — когда разрешим) | → `RESULT` или следующий | Честно показать ответ; CTA дальше по очереди | Блокировать весь прогресс |
| Нет сети на поиске названий | `GUESSING` | Понятная ошибка; можно закрыть ввод и смотреть картинку | Пустой экран |
| Случайно закрыл поле ввода | `VIEWING_FRAGMENT` | Просто снова **Угадать фильм** | Потерять открытые фрагменты |

**Почему это важно:** Engine и UX обязаны вести себя одинаково в «грязных» сессиях — большинство игроков так и играют.

*Уточнение по сдаче и авто-фрагменту при ошибке — открытые Q1/Q2 в v2-game-loop; до решения не делать сдачу главной кнопкой.*

## 8. Success Criteria

### После первого фильма пользователь должен:

1. Понимать правила без отдельного обучения.  
2. Самостоятельно открыть **второй** фильм.  
3. Не задавать вопрос «что делать дальше?».  
4. Понимать, что можно угадать рано или открыть ещё фрагмент.  
5. Не искать календарь / «сегодняшний» уровень.

### После возврата через несколько дней:

1. Увидеть **Продолжить**, не « начать с нуля».  
2. Не почувствовать наказание за пропуск.  
3. За один тап снова оказаться на своём уровне.

### Критерий провала UX

Если в интервью снова звучит «а что мне открыть?» или «я не понял правила» после первого уровня — править экраны, не добавлять туториал.

## 9. Карта вопросов (сводка)

| Экран | Вопросы | Закрываем |
|-------|---------|-----------|
| Первый запуск | Что это? Что делать? | 3 строки + **Играть** |
| Главный | Где прогресс? | **Продолжить** |
| Уровень | Можно отвечать? Мало видно? | Две кнопки |
| Ошибка | Проиграл? | «Неверно» + те же кнопки |
| Фрагмент | Стало яснее? | Новый кусок |
| Результат | Что дальше? | **CTA продолжения** (лейбл TODO) |
| Возврат | С чего? | **Продолжить** |

## 10. Идеальный первый час

1. Три строки → **Играть**  
2. Фрагмент → ещё фрагмент → **«Я понял!»** → угадал  
3. **CTA продолжения** → новый фильм  
4. Ещё 2–3 фильма без выбора дней  
5. Ушёл  
6. Через дни → **Продолжить**

## 11. Design Validation Checklist

Перед любым изменением UX пройти список. Если на вопрос «да, мешает» — упростить, не усложнять объяснениями.

### Можно ли упростить?

- [ ] Можно ли убрать ещё один экран?
- [ ] Можно ли убрать ещё одну кнопку?
- [ ] Можно ли убрать ещё один текст?
- [ ] Можно ли заменить два шага одним?
- [ ] Нужен ли этот элемент, чтобы выполнить **цель** экрана? (если нет — убрать)

### Думает ли игрок вместо игры?

- [ ] Есть ли место, где игрок начинает думать «что нажать?», а не «какой фильм?»
- [ ] Есть ли выбор, который не про угадывание фильма?
- [ ] Есть ли экран без одного очевидного следующего действия?

### Требует ли интерфейс объяснений?

- [ ] Есть ли место, где интерфейс требует пояснения модератором / туториалом?
- [ ] Есть ли технические слова на кнопках?
- [ ] Появился ли календарь, архив или режим на главном пути?

### Соответствие философии

- [ ] Остаётся ли верным: «никогда не думать, что делать дальше»?
- [ ] После победы хочется ли нажать **следующий фильм** без конкурентов?
- [ ] Вернувшись через неделю, продолжит ли игрок с того же места одной кнопкой?

### State Machine

- [ ] Понятно ли, в каком состоянии игрок на этом экране?
- [ ] Каждый ли переход вызван действием игрока или ясной победой — а не «магическим» скачком?
- [ ] Обработаны ли ошибка, уход и возврат без сброса прогресса?

**Правило:** не проходит чеклист — изменение не готово к реализации.
